# Journal Ranking Project

Ranks academic journals by the fraction of papers authored by researchers at elite institutions ("elite revealed preference"), using OpenAlex S3 snapshot data.

## Key Files

- `snapshot/extract_works.py` — S3 extraction pipeline (streams → parquet part files)
- `snapshot/analyze.py` — Analysis: parquet → ranking_data.json
- `snapshot/requirements.txt` — Python dependencies
- `ranking_data.json` — Website data (generated by analyze.py, committed to git)
- `index.html`, `styles.css`, `script.js` — Website (GitHub Pages)
- `agents.md` — Comprehensive project context for AI agents
- `openalex-fields.md` — OpenAlex field reference with sizes and extraction status
- `methodology.md` — Analysis methodology (also used as website content)

## Running

```bash
# Extract data from S3 (6-12 hours for full dataset)
cd snapshot && python3 extract_works.py --workers 8

# Run analysis on extracted data
cd snapshot && python3 analyze.py

# Check extraction status
cd snapshot && python3 extract_works.py --status
```

## Doc Sync Rule

When the extraction schema, institution tiers, filtering criteria, or analysis methodology change, update ALL of the following:
1. `agents.md` — schema section, status section, tier tables
2. `openalex-fields.md` — "Extracting?" column for affected fields
3. `methodology.md` — filtering or tier descriptions
4. Re-run `analyze.py` to regenerate `ranking_data.json`
5. Test the website with the updated data
